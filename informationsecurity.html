▼Lab 1: MD5-Collision-Attack

Objective:The lab assignment focused on understanding and exploiting the weaknesses of the MD5 hashing algorithm, particularly its collision resistance property
Lab Tasks:
Task 1: Generating Two Different Files with the Same MD5 Hash:The main objecctiove is to use the md5collgen tool to generate two different binary files that share the same MD5 hash but contain different contents. The task also includes understanding the effects of file length on the hashing process.
Task 2: Understanding MD5’s PropertyThe task explores the property of MD5 that states if two inputs M and N have the same MD5 hash, then adding the same suffix to both will result in the same hash. Students are expected to design an experiment to demonstrate this property.
Task 3: Generating Two Executable Files with the Same MD5 HashNeed to create two versions of a simple C program such that their MD5 hashes are identical, but the programs contain different data (specifically in an array) and behave differently.
Task 4: Making the Two Programs Behave DifferentlyThis task extends Task 3 by requiring students to create two programs that not only have the same MD5 hash but also execute different instructions (benign vs. malicious behavior). The task discusses how to manipulate binary files to achieve this, with one approach involving the use of two arrays whose contents determine whether benign or malicious code is executed.
Summary:This lab is designed to provide hands-on experience with hash functions, demonstrating the practical implications of cryptographic weaknesses in MD5 by enabling students to see the effects of collision attacks firsthand.

▼Lab 2:SQL-Injection

Overview:SQL injection is a code injection technique that exploits vulnerabilities in web applications, particularly when user inputs are not properly validated before being sent to the database. It is one of the most common attacks on web applications and can lead to unauthorized access, data theft, or data manipulation.
Lab Environment:The lab uses a vulnerable web application hosted on a pre-built Ubuntu 16.04 VM. The web application simulates an employee management system where SQL injection vulnerabilities are intentionally present.
Lab Tasks:
Task 1: Familiarization with SQL Statements
Task 2: SQL Injection Attack on SELECT Statement
Task 3: SQL Injection Attack on UPDATE Statement
Task 4: Countermeasure - Prepared Statement
Key Concepts:
SQL Injection:A vulnerability that allows attackers to execute arbitrary SQL code by manipulating input data.
Prepared Statements:A defense mechanism where SQL code is precompiled with placeholders, ensuring user inputs are treated as data and not executable code.
Summary:By completing this lab, you will gain practical experience in identifying and exploiting SQL injection vulnerabilities, as well as implementing effective countermeasures to protect web applications from such attacks.

▼Lab 3: Secret-Key-Encryption

Objective:To understand secret-key encryption concepts, including algorithms, encryption modes, and initialization vectors (IVs), through a series of practical tasks.
Technologies Used:openssl for encryption and decryption, hex editors for binary manipulation, and frequency analysis tools for deciphering monoalphabetic ciphers.
Key Features:
Frequency Analysis: Decoded a monoalphabetic cipher using frequency analysis.
Encryption Tasks: Experimented with various ciphers and modes (e.g., AES, CBC, CFB, ECB), analyzing encryption results and padding.
Image Encryption: Compared ECB and CBC modes by encrypting and examining BMP images.
Padding Exploration: Investigated PKCS#5 padding by encrypting files of varying sizes and observing padding effects.
Error Propagation: Analyzed the impact of single-bit corruption on data recovery across different encryption modes.
IV Management: Tested the effects of IV uniqueness and predictability on encryption security.
Summary:Acquired practical skills in encryption, addressing tasks related to cipher decoding, encryption mode comparison, padding analysis, error propagation, and IV security to enhance understanding of cryptographic principles and data protection.
